# メニューポップアップのデザイン案

## 🎯 ポップアップ形式のメリット

- ✅ **元の画面が見える** - コンテキストを失わない
- ✅ **軽量で素早い** - 全画面モーダルより軽い
- ✅ **モバイルでも使いやすい** - 画面を占有しすぎない
- ✅ **直感的** - ボタンの近くに表示される

## 📱 推奨実装：サイドパネル形式

### デザイン仕様

既存のテーマモーダルと同様のスタイルで、右側からスライドインする形式：

```
┌─────────────────────────────────┐
│ ヘッダー                        │
│ [ロゴ]  [購入] [ログイン] [☰]  │
└─────────────────────────────────┘
                                    ┌──────────────┐
                                    │ 📊 学習統計  │
                                    │ 🔥 連続学習  │
                                    │ 🎨 背景テーマ│
                                    │ 📱 アプリ追加│
                                    │ ❓ ヘルプ    │
                                    └──────────────┘
```

### 実装詳細

#### スタイル
- **背景**: 半透明の黒（`rgba(0,0,0,0.5)`）+ ブラー効果
- **パネル**: 右側からスライドイン、幅320px（モバイルは90vw）
- **アニメーション**: スムーズなスライドイン/アウト（0.3s）
- **z-index**: テーマモーダルと同じ（z-50）

#### 動作
- メニューボタンクリック → パネルを右からスライドイン
- 背景クリック or 閉じるボタン → パネルを閉じる
- エスケープキー → パネルを閉じる
- 各メニュー項目クリック → 対応する機能を実行

## 🎨 既存のテーマモーダルとの統一

- 同じ背景スタイル（`bg-black/50 backdrop-blur-sm`）
- 同じアニメーション（スライドイン）
- 同じ閉じる方法（背景クリック、×ボタン、エスケープキー）
