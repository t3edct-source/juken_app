# メニューボタンにまとめる要素の提案

## 📋 現在のヘッダー構成

現在、ヘッダーには以下のボタンがあります：
1. **🎨 テーマ** - 背景テーマ選択
2. **📱 アプリに追加** - PWAインストール（条件付き表示）
3. **💳 購入** - コンテンツ購入
4. **ログイン/ログアウト** - 認証

## 🎯 メニューボタンにまとめる要素の候補

- **⚙️ 設定**（将来的に追加可能）

  - 表示設定

### 2. **学習情報・統計系**（推奨度：⭐⭐⭐⭐）
- **📊 学習統計**

  - 累計正解数
  - 平均正解率
  - 学習した単元数
  - 今週の学習時間
- **🔥 連続学習記録**
  - 連続学習日数
  - 現在のレベル
  - アンロック済みテーマ一覧
  - 次のレベルアップまで

### 3. **アカウント・認証系**（推奨度：⭐⭐⭐）
- **👤 アカウント情報**（ログイン時のみメニューに表示）
  - ユーザー名表示
  - メールアドレス表示
  - アカウント設定へのリンク（将来）
- **🔐 パスワード変更**（ログイン時のみ、将来実装）
- ※ **ログイン/ログアウトはヘッダーに残す**（重要な機能のため）

### 4. **補助機能系**（推奨度：⭐⭐⭐）
- **📱 アプリに追加**
  - PWAインストール機能
  - 条件付き表示（インストール可能な場合のみ）
- **❓ ヘルプ・使い方**
  - アプリの使い方
  - よくある質問
  - お問い合わせ
- **📢 お知らせ**（将来的に）
  - 新機能の紹介
  - メンテナンス情報

### 5. **その他**（推奨度：⭐⭐）
- **🔄 データのリセット**
  - 学習履歴のリセット
  - キャッシュのクリア
- **📤 データのエクスポート**（将来的に）
- **📥 データのインポート**（将来的に）

## 💡 実装のポイント

### メニューボタンのデザイン
- **アイコン**: ☰（ハンバーガーメニュー）または ⚙️（設定アイコン）
- **位置**: ヘッダー右端
- **スタイル**: シンプルなアイコンボタン

### メニューモーダル/ドロップダウン
- **表示形式**: 
  - モバイル: 全画面モーダル
  - デスクトップ: 右下からスライドインするサイドパネル
- **セクション分け**: 視覚的に区切りを入れる
- **アイコン**: 各項目に絵文字アイコンを付ける

### 重要な機能の扱い
- **購入ボタン**: メニューに入れるか、ヘッダーに残すか検討
  - メニューに入れる場合: 目立つ位置（上部）に配置
  - ヘッダーに残す場合: メニューにも重複して表示

## 🎯 推奨実装（修正版）

**メニューに含める要素**:
1. ✅ **📊 学習統計** - 新機能として追加
2. ✅ **🔥 連続学習記録** - 詳細情報を表示
3. ✅ **🎨 背景テーマ** - 現在のテーマボタンを移動
4. ✅ **📱 アプリに追加** - 条件付き表示
5. ✅ **❓ ヘルプ** - 新機能として追加

**ヘッダーに残す要素**:
- **💳 購入** - 重要な機能なのでヘッダーに残す
- **ログイン/ログアウト** - 重要な機能なのでヘッダーに残す（常にアクセス可能に）

**理由**:
- ログイン/ログアウトは認証状態を確認・変更する重要な機能
- ユーザーが常にアクセスできるべき
- メニューに隠すと発見しにくくなる
- ヘッダーに表示することで、認証状態が一目で分かる
